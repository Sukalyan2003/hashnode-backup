In the last two posts on this topic, we went through my journey finishing CS50 AI.

But as i’ve said, just passing the test cases is not enough. There are a lot of cool design decisions that they took when making these projects and there is a lot of value in analyzing the code we usually take for granted.

In the process, maybe we can make those projects do some related cool things. Let’s begin.

**The structure would be, explain the boilerplate in some detail for each of the projects; List down some interesting ways to modify them; then implement one or more of those proposed modifications and show the results.**
